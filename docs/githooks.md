# Git Hooks Standard

This document defines the official `brainxio/cicd` git hook standards that all template repositories must follow.

## Philosophy

Git hooks enforce quality gates **before** code reaches the remote repository. They catch errors, style violations, and security issues at commit-time or push-time, preventing broken code from entering the shared history.

> Hooks are local. They supplement — not replace — CI checks. A commit that passes hooks should also pass CI.

## Installation

Copy the `.githooks/` directory from `brainxio/cicd` into your repository root:

```bash
# Clone brainxio/cicd as a submodule
mkdir -p .git/modules
git submodule add https://github.com/BrainXio/cicd.git .git/modules/cicd

# Symlink hooks (preferred — gets updates automatically)
ln -s .git/modules/cicd/.githooks/* .githooks/

# Or copy hooks (static snapshot)
cp -r .git/modules/cicd/.githooks/* .githooks/
```

Ensure hooks are executable:

```bash
chmod +x .githooks/*
```

Configure git to use the local hooks directory:

```bash
git config core.hooksPath .githooks
```

## Hook Reference

### `commit-msg`

**When it runs:** After the commit message is written but before the commit is created.

**What it does:**
- Validates [Conventional Commits](https://www.conventionalcommits.org/) format: `type(scope): description`
- Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `ci`, `build`, `perf`
- Rejects AI attribution patterns:
  - `Co-authored-by`
  - `Generated by AI`
  - `AI-generated`
  - References to Claude, ChatGPT, Copilot, Gemini, Grok, LLaMA

**Exit codes:**
- `0` — Message is valid
- `1` — Format violation or forbidden attribution found

### `pre-commit`

**When it runs:** After `git add` but before the commit is created.

**What it validates (staged files only):**
- **YAML syntax** — All `.yaml`/`.yml` files parse correctly (skips `.github/`)
- **Jinja2 templates** — Templates compile without syntax errors
- **Namespace restriction** — Repository must belong to an allowed namespace (default: `brainxio`)
- **Trailing whitespace** — Blocks commits with trailing spaces

**Allowed namespaces:**

Defined as a bash array at the top of the hook:

```bash
ALLOWED_NAMESPACES=("brainxio")
```

Add more namespaces as needed:

```bash
ALLOWED_NAMESPACES=("brainxio" "my-org" "client-org")
```

**Exit codes:**
- `0` — Staged files pass validation
- `1` — One or more validation errors found

### `pre-push`

**When it runs:** After `git push` but before refs are updated on the remote.

**What it validates (full repository scan):**

| Check | Description | Severity |
|-------|-------------|----------|
| **AI Attribution Scan** | Scans commit history for AI attribution patterns | Warning |
| **Secret Detection** | Detects API keys, tokens, passwords, private keys in staged files | Warning |
| **Merge Conflict Markers** | Blocks if `<<<<<<<`, `=======`, or `>>>>>>>` markers exist | Error |
| **Trailing Whitespace** | Blocks files with trailing whitespace | Error |
| **Large Files** | Warns on files > 1MB | Error |
| **Binary Files** | Warns on unexpected binary files (executables, archives, media) | Warning |
| **Executable Bits** | Warns when non-script files have executable permissions | Warning |
| **YAML Validation** | Parses all YAML files with pyyaml | Error |
| **Jinja2 Validation** | Compiles all `.jinja`/`.jinja2`/`.j2` files and embedded templates | Error |
| **Python Syntax** | Validates `.py` files with `py_compile` | Error |
| **Shell Syntax** | Validates `.sh` files with `bash -n` | Error |
| **JSON Syntax** | Validates `.json` files | Error |
| **Go Syntax** | Validates `.go` files with `gofmt` (if available) | Error |
| **Rust Syntax** | Validates `.rs` files with `rustc` (if available) | Error |
| **TypeScript/JavaScript** | Lints `.ts`/`.tsx`/`.js`/`.jsx` files with `eslint` (if available) | Warning |

**Severity levels:**
- **Error** — Push is blocked. Must fix before retrying.
- **Warning** — Push proceeds, but warnings are displayed.

## Customization

### Adding a New Namespace

Edit `.githooks/pre-commit`:

```bash
ALLOWED_NAMESPACES=("brainxio" "your-namespace")
```

### Adding a New File Type Check

Edit `.githooks/pre-push` and add a new validation block before the `SUMMARY` section:

```bash
# Your language validation
if git diff --cached --name-only --diff-filter=ACM | grep -q '\.ext$'; then
  echo "Running .ext validation..."
  while IFS= read -r -d '' file; do
    if ! your-validator "$file" 2>/dev/null; then
      echo "ERROR: Validation failed for $file" >&2
      errors=$((errors + 1))
    fi
  done < <(git diff --cached --name-only --diff-filter=ACM -z | grep -z '\.ext$' || true)
fi
```

### Skipping Hooks (Emergency Only)

```bash
# Skip pre-commit (not recommended)
git commit --no-verify -m "emergency fix"

# Skip pre-push (not recommended)
git push --no-verify
```

> **Warning:** `--no-verify` bypasses all quality gates. Only use for genuine emergencies and fix the underlying issue immediately.

## Troubleshooting

### "Namespace not allowed" Error

Your repository remote URL does not match any namespace in `ALLOWED_NAMESPACES`. Check:

```bash
git remote get-url origin
```

Expected format: `https://github.com/BRAINXIO/repo.git` or `git@github.com:brainxio/repo.git`

### Hook Not Running

1. Verify `core.hooksPath` is set:
   ```bash
   git config core.hooksPath
   # Should print: .githooks
   ```

2. Verify hooks are executable:
   ```bash
   ls -la .githooks/
   # Should show executable bits (x)
   ```

3. If using symlinks, verify the target exists:
   ```bash
   ls -la .githooks/pre-commit
   # Should show arrow pointing to real file
   ```

### CI Passes but Hooks Fail

Hooks run stricter checks than CI in some cases (e.g., namespace validation). This is by design. Fix the local issue — the hook is catching something CI does not check.
